Upgrade flow and steps
```
1. CPD 5.2.1 pre-check
2. Update cpd-cli and environment variables script for 5.3.1
3. Upgrade shared cluster components (ibm-licensing, ibm-scheduler)
4. Prepare to upgrade an instance of IBM Software Hub
5. Upgrade an instance of IBM Software Hub
6. Upgrade CPD services (datastage_ent,ws_pipelines)
7. Apply hot fixes (CCS, Pipelines, DataStage)
8. Upgrade the cpdbr service
9. Validate CPD upgrade (customer acceptance test)
```

### Backing up your existing certificates

Before you uninstall the IBM Certificate manager, back up the Issuer and Certificate custom resources

Create a temporary project where you can validate that the IBM Certificate manager is working correctly
```
oc new-project cert-mgr-test
```

Back up the Issuer custom resources to a file named issuer_list.yaml
```
oc get issuers.cert-manager.io -A -o yaml > issuer_list.yaml
```

Back up the Certificate custom resources to a file named certificate_list.yaml
```
oc get certificates.cert-manager.io -A -o yaml > certificate_list.yaml
```

Verify that IBM Certificate manager is working correctly

Create an Issuer custom resource called verify-issuer
```
cat <<EOF |oc apply -f -
apiVersion: cert-manager.io/v1
kind: Issuer
metadata:
  name: verify-issuer
spec:
  selfSigned: {}
EOF
```

Apply the contents of the issuer_list.yaml file
```
oc apply -f issuer_list.yaml
```

Create a Certificate custom resource called verify-certificate
```
cat <<EOF |oc apply -f -
apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: verify-certificate
spec:
  commonName: verify-certificate
  secretName: verify-secret
  issuerRef:
    name: verify-issuer
    kind: Issuer
    group: cert-manager.io
EOF
```

Apply the contents of the certificate_list.yaml file
```
oc apply -f certificate_list.yaml
```

Verify that the verify-certificate custom resource is ready
```
oc get issuers.cert-manager.io
```

### Uninstall IBM Certificate manager 

Before you can install the Red Hat® OpenShift® certificate manager (cert-manager Operator), you must uninstall the IBM Certificate manager

Get the list of the IBM Certificate manager configuration instances
```
oc get certmanagerconfig \
-n ${PROJECT_CERT_MANAGER}
```

Delete each configuration instance returned by the preceding command
```
oc delete certmanagerconfig <name> \
-n ${PROJECT_CERT_MANAGER}
```

Uninstall the IBM Certificate manager operator

Delete the operator subscription:
```
oc delete sub ibm-cert-manager-operator \
-n ${PROJECT_CERT_MANAGER}
```

Find any ibm-cert-manager cluster service versions (CSVs)
```
oc get csv \
-n ${PROJECT_CERT_MANAGER} \
| grep ibm-cert-manager
```

Delete the CSVs returned by the previous command
```
oc delete csv <name> \
-n ${PROJECT_CERT_MANAGER}
```

Verify that the following IBM Certificate manager resources are deleted

Check for any deployments with the app.kubernetes.io/component=cert-manager label
```
oc get deployments \
-n ${PROJECT_CERT_MANAGER} \
-l app.kubernetes.io/component=cert-manager
```

If any deployments are returned by the preceding command, delete them
```
oc delete deployments <name> \

oc get service \

If any services are returned by the preceding command, delete them

Check for any cert-manager-webhook mutating web hook configurations
```

-n ${PROJECT_CERT_MANAGER} \

If any mutating web hook configurations are returned by the preceding command, delete them

## 2. Update cpd-cli, Helm CLI and environment variables script for 5.3.1

wget https://github.com/IBM/cpd-cli/releases/download/v14.3.1/cpd-cli-linux-EE-14.3.1.tgz && gzip -d cpd-cli-linux-EE-14.3.1.tgz && tar -xvf cpd-cli-linux-EE-14.3.1.tar && rm -rf cpd-cli-linux-EE-14.3.1.tar

Add the cpd-cli to your PATH variable
```
export PATH=/root/cpd-cli-linux-EE-14.3.1-2986:$PATH

```
vi cpd_vars.sh

Update the Version field and save your changes
```
VERSION=5.3.1
```

Set your $IMAGE_PULL_SECRET, $IMAGE_PULL_CREDENTIALS and IMAGE_PULL_PREFIX environment variables

export IMAGE_PULL_CREDENTIALS=$(echo -n "$PRIVATE_REGISTRY_PULL_USER:$PRIVATE_REGISTRY_PULL_PASSWORD" | base64 -w 0)

source cpd_vars.sh

## 5. Upgrade an instance of IBM Software Hub (est. 30 minutes)

### Create image pull secrets for an instance of IBM Software Hub:

You must create an image pull secret in the projects that are associated with an instance of IBM Software Hub

The image pull secrets ensure that any components that use Helm have access to the credentials for pulling images

Authenticate oc-cli
```
${OC_LOGIN}
```

Set your $IMAGE_PULL_SECRET and $IMAGE_PULL_CREDENTIALS environment variables
```
export IMAGE_PULL_SECRET=<pull-secret-name>
export IMAGE_PULL_CREDENTIALS=$(echo -n "$PRIVATE_REGISTRY_PULL_USER:$PRIVATE_REGISTRY_PULL_PASSWORD" | base64 -w 0)
export IMAGE_PULL_PREFIX=${PRIVATE_REGISTRY_LOCATION}
```

Create a file named dockerconfig.json based on where your cluster pulls images from

For private container registry
```
cat <<EOF > dockerconfig.json 
{
  "auths": {
    "PRIVATE_REGISTRY_LOCATION": {
      "auth": "${IMAGE_PULL_CREDENTIALS}"
    }
  }
}
EOF
```

Create the image pull secret in the operators project for the instance:
```
oc create secret docker-registry ${IMAGE_PULL_SECRET} \
--from-file ".dockerconfigjson=dockerconfig.json" \
--namespace=${PROJECT_CPD_INST_OPERATORS}
```

Create the image pull secret in the operands project for the instance:
```
oc create secret docker-registry ${IMAGE_PULL_SECRET} \
--from-file ".dockerconfigjson=dockerconfig.json" \
--namespace=${PROJECT_CPD_INST_OPERANDS}
```

Now that you've created the image pull secrets for this instance of IBM Software Hub, you're ready to upgrade IBM Software Hub

### [Upgrade the required operators and custom resources for the instance](https://www.ibm.com/docs/en/software-hub/5.3.x?topic=uish-upgrading-software-hub):

Log the cpd-cli in to the Red Hat® OpenShift® Container Platform cluster:

```

#######################################
# Query cluster for CCS resources and select namespace
#######################################
prompt_for_namespace() {
 info_msg "Querying cluster for CCS resources..."

 # Get all namespaces that contain CCS resources
 local ccs_namespaces
 local query_output
 local query_exit_code

 query_output=$(oc get ccs --all-namespaces --no-headers 2>&1)
 query_exit_code=$?

 if [[ $query_exit_code -ne 0 ]]; then
 error_exit "Failed to query CCS resources (exit code: $query_exit_code)\nError:
$query_output"
 fi

 ccs_namespaces=$(echo "$query_output" | awk '{print $1}' | sort -u)
 
 if [[ -z "$ccs_namespaces" ]]; then
 error_exit "No CCS resources found in the cluster. Please ensure CP4D is installed."
 fi

 # Convert to array
 local namespaces_array=()
 while IFS= read -r line; do
 if [[ -n "$line" ]]; then
 namespaces_array+=("$line")
 fi
 done <<< "$ccs_namespaces"

 # Handle based on number of namespaces found
 if [[ ${#namespaces_array[@]} -eq 1 ]]; then
 # Only one namespace - use it automatically
 NAMESPACE="${namespaces_array[0]}"
 success_msg "Found CCS resource in namespace:
${COLOR_BOLD}$NAMESPACE${COLOR_RESET}"
 else
 # Multiple namespaces - present picker
 echo ""
 echo -e "${COLOR_BOLD}Found CCS resources in multiple
namespaces:${COLOR_RESET}"
 for i in "${!namespaces_array[@]}"; do
 echo -e " ${COLOR_BOLD}$((i + 1)))${COLOR_RESET} ${namespaces_array[i]}"
 done

 echo ""
 echo -e "${COLOR_CYAN}Please select a namespace (enter
number):${COLOR_RESET}"
 read -r choice

 if ! [[ "$choice" =~ ^[0-9]+$ ]] || [[ "$choice" -lt 1 ]] || [[ "$choice" -gt
${#namespaces_array[@]} ]]; then
 error_exit "Invalid selection. Please enter a number between 1 and
${#namespaces_array[@]}"
 fi

 NAMESPACE="${namespaces_array[$((choice - 1))]}"
 success_msg "Selected namespace:
${COLOR_BOLD}$NAMESPACE${COLOR_RESET}"
 fi

 # Verify namespace is accessible
 if ! oc get project "$NAMESPACE" &>/dev/null; then
 error_exit "Namespace '$NAMESPACE' exists but is not accessible"
 fi
}

#######################################
# Build OpenSearch query for glossary artifacts
# Arguments:
# $1 - include_size_and_tracking (true/false) - whether to include size and
track_total_hits
# Returns:
# Query JSON string
#######################################
build_glossary_query() {
 local include_size_and_tracking="${1:-false}"
 local size_param=""
 local track_hits=""

 if [[ "$include_size_and_tracking" == "true" ]]; then
 size_param="\"size\": 0,"
 track_hits="\"track_total_hits\": true,"
 fi

 cat <<EOF
{
 ${track_hits}
 ${size_param}
 "query": {
 "bool": {
 "must": [
 { "term": { "provider_type_id": "glossary" } }${ARTIFACT_TYPE_SELECTOR}
 ]
 }
 }
}
EOF
}

 -XPOST

 --header 'content-type: application/json' \
 --data "$query" 2>&1)

 local http_code=$(echo "$response" | tail -n1)
 local body=$(echo "$response" | sed '$d')

 echo "$body" | jq '.' 2>/dev/null || echo "$body"
 success_msg "Delete operation initiated"
 else
 echo -e "${COLOR_RED}Delete operation failed with HTTP status:
$http_code${COLOR_RESET}"
 echo "$body" | jq '.' 2>/dev/null || echo "$body"
 error_exit "Failed to delete artifacts from OpenSearch"
 fi
}

#######################################
# Wait for resync to stabilize
#######################################
wait_for_stabilization() {
 info_msg "Monitoring resync progress (timeout: ${MAX_WAIT_MINUTES} minutes)..."

 local previous_count=-1
 local stable_iterations=0
 local start_time=$(date +%s)
 local max_wait_seconds=$((MAX_WAIT_MINUTES * 60))
 local iteration=0

 while true; do
 local current_time=$(date +%s)
 local elapsed=$((current_time - start_time))
 local elapsed_min=$((elapsed / 60))
 local elapsed_sec=$((elapsed % 60))

 # Check timeout
 if [[ $elapsed -gt $max_wait_seconds ]]; then
 echo ""
 warning_msg "Timeout reached after ${MAX_WAIT_MINUTES} minutes. Final
count: $previous_count"
 warning_msg "Sync may still be in progress. Monitor manually if needed."
 break
 fi

 local current_count
 current_count=$(get_existing_artifacts_count)
 iteration=$((iteration + 1))

 # Calculate rate if we have previous data
 local rate_info=""
 if [[ $previous_count -ge 0 ]] && [[ $previous_count -ne $current_count ]]; then
 local diff=$((current_count - previous_count))
 local rate=$(echo "scale=1; $diff / $STABILIZATION_WAIT_SECONDS" | bc
2>/dev/null || echo "N/A")
 if [[ "$rate" != "N/A" ]]; then
 rate_info=" (${diff:+"+"}$diff docs, ${rate}/sec)"
 fi
 fi

 # Check for stability
 if [[ "$current_count" -eq "$previous_count" ]]; then
 stable_iterations=$((stable_iterations + 1))
 echo -e "${COLOR_CYAN} [$elapsed_min:$(printf "%02d" $elapsed_sec)] Count:
$current_count (stable:
$stable_iterations/$STABILIZATION_ITERATIONS)${COLOR_RESET}"
 else
 stable_iterations=0
 echo -e "${COLOR_CYAN} [$elapsed_min:$(printf "%02d" $elapsed_sec)] Count:
$current_count (was: $previous_count)$rate_info${COLOR_RESET}"
 fi

 # Check if stabilized
 if [[ "$stable_iterations" -eq "$STABILIZATION_ITERATIONS" ]]; then
 echo ""
 success_msg "Synchronization completed in ${elapsed_min}m ${elapsed_sec}s,
count stabilized at ${COLOR_BOLD}$current_count${COLOR_RESET}"
 break
 fi

 previous_count=$current_count
 sleep "$STABILIZATION_WAIT_SECONDS"
 done
}

```

--component=cpdbr-tenant \
--cpdbr-hooks-image-prefix=${PRIVATE_REGISTRY_LOCATION}/cpopen/cpfs \

--cpd-scheduler-namespace=${PROJECT_SCHEDULING_SERVICE} \

--upgrade=true \
--log-level=debug \
